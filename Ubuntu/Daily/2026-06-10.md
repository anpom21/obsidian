Answer:
Yes. The easiest way is to use Nautilus Scripts.

Create an executable script in:

~/.local/share/nautilus/scripts/

Anything executable in that folder appears in Nautilus under:

Right click → Scripts

GNOME’s docs confirm that scripts in this directory are shown in the file context menu, and selected files are passed to the script.  

Example Python script:

mkdir -p ~/.local/share/nautilus/scripts

nano ~/.local/share/nautilus/scripts/MyPythonAction

chmod +x ~/.local/share/nautilus/scripts/MyPythonAction

Put this inside:

#!/usr/bin/env python3

  

import os

import subprocess

from pathlib import Path

  

selected = os.environ.get("NAUTILUS_SCRIPT_SELECTED_FILE_PATHS", "")

paths = [Path(p) for p in selected.splitlines() if p]

  

# If nothing is selected, fall back to current folder

current_dir = os.environ.get("NAUTILUS_SCRIPT_CURRENT_URI", "")

  

message = "Selected:\n" + "\n".join(str(p) for p in paths)

  

subprocess.run([

    "zenity",

    "--info",

    "--title=Nautilus Python Script",

    f"--text={message or 'No selected files'}"

])

Then restart Nautilus:

nautilus -q

Open Nautilus again, right-click a file/folder, and you should see:

Scripts → MyPythonAction

Useful environment variables include NAUTILUS_SCRIPT_SELECTED_FILE_PATHS, which contains the selected local file/folder paths, usually newline-separated.  

For a folder-specific action, right-click the folder itself. Your Python script can then read the selected path:

#!/usr/bin/env python3

  

import os

from pathlib import Path

  

selected = os.environ.get("NAUTILUS_SCRIPT_SELECTED_FILE_PATHS", "")

folders = [Path(p) for p in selected.splitlines() if p]

  

for folder in folders:

    if folder.is_dir():

        print(f"Do something with {folder}")

One caveat: this appears under the Scripts submenu, not usually as a top-level right-click item. For a true custom top-level context-menu entry, use a Nautilus extension via nautilus-python, but for most quick automations the scripts folder is much easier.
